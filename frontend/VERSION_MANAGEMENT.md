# Управление версиями приложения

## Проблема

Ранее в проекте было несколько источников версий, что приводило к рассинхронизации:
- `package.json` - основная версия
- `.env` файл - переменные `APP_VERSION_NAME` и `APP_VERSION_CODE`
- Android `build.gradle` - использовал переменные окружения

## Решение

Теперь **единственным источником правды** является `package.json`. Все остальные источники автоматически синхронизируются с ним.

## Скрипты управления версиями

### Основные команды

```bash
# Показать текущую версию
npm run version:current

# Установить конкретную версию
npm run version:set 2.0.48

# Увеличить версию
npm run version:bump:patch    # 2.0.47 -> 2.0.48
npm run version:bump:minor    # 2.0.47 -> 2.1.0
npm run version:bump:major    # 2.0.47 -> 3.0.0

# Синхронизировать версию из package.json в .env
npm run version:env
```

### Сборка приложения

```bash
# Простая сборка frontend (без изменения версии)
npm run build

# Сборка APK с АВТОМАТИЧЕСКИМ увеличением версии
npm run build-apk             # Автоматически +1 к patch версии и сборка APK

# Сборка с УПРАВЛЯЕМЫМ увеличением версии
npm run build:bump patch      # Увеличить patch и собрать APK
npm run build:bump minor      # Увеличить minor и собрать APK
npm run build:bump major      # Увеличить major и собрать APK

# Или использовать скрипт напрямую
./build-with-version-bump.sh patch
./build-with-version-bump.sh 2.0.48

# СТАРЫЙ СКРИПТ (тоже работает, но рекомендуется build:bump)
npm run build-auto patch      # Увеличить patch и собрать APK
./build-and-bump.sh patch     # То же самое, но напрямую
```

## Как это работает

1. **version-manager.js** - читает версию из `package.json`
2. **Синхронизация** - `npm run version:env` обновляет `.env` файл
3. **Сборка APK** - `build-apk.sh` автоматически синхронизирует версию перед сборкой
4. **Имя APK** - формируется из версии из `package.json`

## Формат имени APK

```
pl-price-{VERSION}-{YYMMDD-HHMM}.apk
```

Например: `pl-price-2.0.47-260402-1123.apk`

## Важные изменения

- Убрано автоматическое увеличение версии из скрипта `build`
- **Добавлено автоматическое увеличение версии в `build-apk`** (+1 к patch при каждой сборке)
- Добавлена принудительная синхронизация версии перед сборкой APK
- Создан новый скрипт `build-with-version-bump.sh` для сборки с управляемым увеличением версии

## Рекомендации

1. **Для быстрой разработки**: используйте `npm run build` (без изменения версии)
2. **Для автоматических сборок**: используйте `npm run build-apk` (авто +1 к версии)
3. **Для контролируемых релизов**: используйте `npm run build:bump patch/minor/major`
4. **Перед сборкой**: всегда проверяйте версию `npm run version:current`
5. **Для принудительной синхронизации**: `npm run version:env`
